# Folder drag-and-drop monitoring
_Exported on 7/12/2026 at 16:11:36 PDT from Cursor (3.11.13)_

---

**User**

When a folder is opened as a result of a drag-and-drop operation, file change monitoring must be enabled for that folder.
This has already been implemented for drag-and-drop operations when opening a folder, but it has not been implemented for opening a folder via an .lnk file.

---

**Cursor**

I'll trace how drag-and-drop enables folder monitoring and how `.lnk` opens folders, then align the `.lnk` path with the same behavior.


I see the gap: drag-and-drop folders attach `FileSystemFileHandle`s so monitoring works, but `.lnk` opens load bytes without handles and end up as transient sources. I'll add disk-path sources for backend-loaded files and wire them into the monitor/reload pipeline.


The monitor tick was passing size-only stubs into reload — I'll separate size probing from full file reads for path-based sources.


Opening a folder via drag-and-drop already worked for monitoring because dropped files get `FileSystemFileHandle`s. Opening via `.lnk` loaded file bytes without any reloadable source, so monitoring could not run. That gap is now closed.

## What changed

**Disk-path file sources** — When files are loaded through the Go backend (including `.lnk` → folder), each file keeps its disk path via `setFileDiskPath`, and `createFileSourceInfo` creates `{ kind: "path", path }` sources instead of transient ones.

**Backend `StatPaths`** — A lightweight Go method returns file sizes for the monitor tick without re-reading full file contents on every poll.

**Monitor/reload pipeline** — `canReloadFile`, `asyncMonitorTick`, and `parseLatestFileWithRetry` now support both `handle` and `path` sources. Path-based files are re-read through `readFileFromBackendPath` when content reload is needed.

**Auto-enable monitoring for folders** — When `asyncLoadAnyFiles` is called with a `droppedFolderName` (folder drop or `.lnk` → folder), `sizeMonitorEnabled` is turned on automatically if any reloadable files were loaded. This applies equally to HTML5 folder drops and `.lnk` shortcuts.
