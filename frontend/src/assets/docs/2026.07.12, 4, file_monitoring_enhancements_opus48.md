## Summary

I moved file-change detection for desktop (path-based) files from frontend polling into a real Windows filesystem watcher in Go, and added new-file / deletion handling with the requested UI.

### Go backend (new Windows FS monitor)
- **`backend/monitor.go`** — bound `StartMonitor(req)` / `StopMonitor()` methods, request/event types (`StartMonitorRequest{folders, files}`, `MonitorChanges{added, modified, deleted}`), and helpers.
- **`backend/monitor_windows.go`** — a real `ReadDirectoryChangesW` watcher (via `golang.org/x/sys/windows`) using overlapped I/O + a stop event for clean cancellation. Each change notification triggers a debounced folder scan that diffs `.trc3` files against a baseline and emits a `monitor-changes` event with precise added / deleted / modified (size or mtime) sets. The baseline is seeded from the folder scan **and** the file sizes the web reports — that's the "sync" of loaded state.
- **`backend/monitor_other.go`** — no-op stub for non-Windows.
- Wired `monitorCancel`/`monitorMu` into the `App` struct and stop the monitor in `BeforeClose`.

### Frontend
- **Go is now the source of truth for desktop files**; the polling `asyncMonitorTick` is narrowed to browser folder handles (which have no disk path).
- **`MonitorControls`** starts/stops the Go watcher for path files (folders derived from the loaded files' parent dirs) and restarts only when the *set* of files changes.
- **`asyncApplyMonitorChanges`** (in `8-1-load-files.ts`): added → load contents + append + green flash; deleted → red flash then remove; modified → reuse the existing size-tracking / tail-follow reload path. Only new `.trc3` files are auto-added (respecting block-load filters).
- **Green/red row flash** in `1-file-list-row.tsx`, driven by new `flashKind`/`flashExpiresAt` fields on `FileUpdateInfo`.
- **`+N` (green) / `-N` (red) notice** added to `3-0-notice-all.tsx`, backed by a new notice store.
- **New option** "Added/removed file highlight duration (ms)" (default 1000) governs both the row flash and the notice.
- Event listener wired via `init-wails-monitor.ts` in `main.tsx`; Wails bindings (`App.js/.d.ts`, `models.ts`, `is-wails.ts`) and a `wails/monitor.ts` wrapper added.

A couple of notes on decisions I made:
- Deletions flash red then the row is removed after the configured duration.
- The `+N/-N` counter accumulates within the highlight window and clears together after the duration.
- Files opened via the browser "Open Folder" menu produce handles without disk paths, so those keep using the polling fallback (Go can't watch a path it doesn't have) — the Go watcher kicks in for the desktop drag-drop / OS-open / folder flows where real paths exist.
